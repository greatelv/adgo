## 1. 제품 개요 및 목표 (Overview & Objectives)

### 1.1. 배경 및 목적

1인 크리에이터와 인디 메이커들은 마케팅 리소스 부족으로 초기 유저 확보에 어려움을 겪습니다. '광고고'는 이들이 필요한 마케팅 액션(가입, 구독, 리뷰 등)을 **'퀘스트(의뢰)'** 형태로 등록하고, 서포터들이 이에 지원하면 빌더가 적합한 인원을 **'선별'**하여 직접 소통하게 돕는 **무료 매칭 플랫폼**입니다.

### 1.2. 핵심 가치 제안 (Value Proposition)

- **Builder (의뢰자):** 비용 부담 없이(수수료 0원) 원하는 서포터를 모집하고, 직접 선별하여 고품질의 마케팅 성과를 확보.
- **Supporter (참여자):** 자신의 SNS/커뮤니티 영향력을 활용하여 다양한 미션에 참여하고 보상을 획득.

### 1.3. MVP 성공 지표 (KPIs)

- **North Star Metric:** **주간 성사된 매칭(Selected) 수 20건**
- **보조 지표:**
    - 퀘스트 등록 전환율 (가입 대비 등록 수)
    - 퀘스트 당 평균 지원자 수 (3명 이상 목표)

## 3. 핵심 기능 명세 (Functional Requirements)

### 3.1. 회원 및 인증 (Auth) 및 프로필

- **인증 방식:** **카카오**, 네이버, 구글 소셜 로그인 (Supabase Auth). *접근성을 위해 카카오 로그인 권장.*
- **프로필 정책:**
    - **연락처 필수 등록:** 마이페이지에서 '매칭용 연락처'를 등록해야 지원 가능.
    - **Privacy First:** 개인 전화번호보다는 **오픈카톡 링크**를 강력히 권장 (가이드 문구 제공).
- **서포터 활동처 (Channels):**
    - **SNS:** 인스타그램, X(트위터), 틱톡, 스레드
    - **메신저/커뮤니티:** 카카오톡, 디시인사이드, 펨코, 인벤 등
    - **플랫폼:** 유튜브, 블로그(네이버/티스토리), 앱스토어

### 3.2. 퀘스트(Quest) 구조 & 보드

- **구조:** `Channel` (활동처) + `Type` (액션 종류) 조합
    - 예: `유튜브` + `구독`, `앱스토어` + `리뷰`, `인스타그램` + `팔로우`
- **등록 프로세스:**
    - **필수 입력:** 제목, 상세 내용, 모집 인원(최대 100명), **마감일(Deadline)**.
    - **보상 정보:** 텍스트 필드 (예: "스타벅스 기프티콘", "편의점 3천원권"). *금전 지금 보장 기능 없음.*
- **리스트 (Board):**
    - 필터: 최신순, 마감임박순, 인기순(조회/지원 많은 순).
    - 상태 배지: `모집중(D-Day)`, `심사중(선정 완료/인원 초과)`, `마감`.

### 3.3. 매칭 워크플로우 (Matching Workflow) - **Core Logic**

본 플랫폼은 **[지원] -> [심사] -> [매칭(공개)]**의 3단계 비동기 커뮤니케이션을 따릅니다.

1.  **지원 (Apply):**
    - 서포터가 퀘스트 확인 후 [지원하기] 클릭.
    - **연락처 확인:** 프로필에 저장된 연락처가 기본값으로 뜨며, 수정 가능 (Snapshot 저장).
    - **각오 한마디:** 빌더에게 어필할 짧은 메시지 작성.
    - **상태:** `APPLIED`

2.  **심사 및 선정 (Select):**
    - 빌더는 [지원자 관리] 페이지에서 지원자 리스트 확인.
    - **Blind:** 선정 전까지 지원자의 구체적인 연락처는 **마스킹 처리** 혹은 **비공개**. (닉네임, 채널, 각오만 공개)
    - **Action:** 맘에 드는 지원자를 [선정하기] 클릭.
    - **상태:** `SELECTED`

3.  **매칭 완료 및 컨택 (Match & Contact):**
    - 선정 즉시 빌더에게 서포터의 **실제 연락처(오픈카톡 링크 등)가 공개됨**.
    - 빌더는 해당 연락처로 개별 연락하여 미션 가이드 전달 및 보상 협의.
    - 플랫폼의 역할은 여기서 종료.

### 3.4. 사후 관리 (Post-Match)

- **완료 처리 (Complete):**
    - 빌더가 외부에서 미션이 완료되었음을 확인하면, 관리 페이지에서 [완료] 처리.
    - 이는 추후 서포터의 '수행 이력(Reputation)' 데이터로 활용됨.
- **신고하기 (Report):**
    - '연락 두절', '약속 불이행', '부적절한 요구' 등의 사유로 상대를 신고하는 기능 (MVP는 운영자 이메일 제보 링크로 대체).

## 4. 데이터베이스 스키마 설계 (Supabase)

*Senior PO Note: 확장성을 고려하되, MVP 속도를 위해 필수 필드 위주로 구성했습니다.*

```sql
-- 1. Users & Profile
create table public.users (
  id uuid references auth.users not null primary key,
  email text not null,
  nickname text,
  avatar_url text,
  marketing_agreed boolean default false, -- 마케팅 동의 여부
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. User Contacts (매칭용 연락처)
create table public.user_contacts (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users not null,
  contact_type text check (contact_type in ('KAKAO_OPENCHAT', 'EMAIL', 'PHONE', 'TELEGRAM')),
  contact_value text not null, -- 실제 링크나 번호
  is_primary boolean default false, -- 기본 연락처 여부
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. Quests (의뢰글)
create table public.quests (
  id uuid default gen_random_uuid() primary key,
  builder_id uuid references public.users not null,
  title text not null,
  description text not null, -- Markdown/HTML content
  category text, -- Deprecated (Use channel/quest_type)
  channel text not null, -- 활동 채널 (Youtube, Instagram...)
  quest_type text not null, -- 활동 유형 (Subscribe, Review...)
  reward_text text, -- 사례금 (Text only)
  recruit_count integer not null default 1, -- 모집 인원
  deadline timestamp with time zone, -- 마감일 (Optional)
  status text default 'RECRUITING' check (status in ('RECRUITING', 'IN_PROGRESS', 'CLOSED', 'CANCELLED')),
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 4. Quest Applications (지원 및 매칭)
create table public.quest_applications (
  id uuid default gen_random_uuid() primary key,
  quest_id uuid references public.quests not null,
  supporter_id uuid references public.users not null,
  appeal_message text, -- 지원 동기/각오
  contact_info_snapshot text not null, -- 지원 시점의 연락처 (수정 불가 Snapshot)
  status text default 'APPLIED' check (status in ('APPLIED', 'SELECTED', 'COMPLETED', 'REJECTED')),
  reject_reason text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  unique(quest_id, supporter_id)
);
```

## 5. 정책 및 안전 장치 (Policies & Safety)

### 5.1. 매칭 정책
- **중복 지원:** 가능.
- **자동 마감:**
    1.  설정된 `deadline` 경과 시.
    2.  `SELECTED` 된 인원이 `recruit_count`에 도달 시 `IN_PROGRESS`(심사완료) 상태로 자동 변경.

### 5.2. 책임의 한계 및 면책 (Disclaimer)
- 모든 퀘스트 상세 페이지 하단 및 지원/선정 팝업에 다음 경고 문구를 명시한다.
    > "광고고는 매칭 중개 플랫폼이며, 실제 미션 수행 및 보상 지급은 당사자 간의 책임 하에 진행됩니다. 금전 요구 등 의심스러운 상황 발생 시 즉시 신고해주시기 바랍니다."

### 5.3. 개인정보 보호
- 서포터의 연락처는 오직 '선정된 빌더'에게만 노출된다.
- 매칭 후 일정 기간(예: 30일)이 지나면 연락처 정보는 마스킹 처리되거나 파기될 수 있음을 고지(약관).

## 6. 개발 우선순위 (Dev Priority)

| Priority | Feature | Note |
| --- | --- | --- |
| **P0 (Critical)** | 회원가입/로그인, 프로필(연락처) 관리 | 이것 없이는 서비스 불가 |
| **P0 (Critical)** | 퀘스트 등록(C), 리스트 조회(R), 상세 조회(R) | 핵심 컨텐츠 |
| **P1 (High)** | 지원하기(Apply), 지원자 관리 및 선정(Select) | 핵심 기능 |
| **P2 (Medium)** | 마이페이지 (내가 쓴 퀘스트, 내가 지원한 퀘스트) | 유저 리텐션 |
| **P3 (Low)** | 알림(Notification), 신고하기, 검색 필터 고도화 | 출시 후 고도화 가능 |
